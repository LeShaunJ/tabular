    Tabular

    Provide go-cobra-style tab-completions for your Crystal CLI. While it is opinionated in it's adherence to the go-cobra convention, it is unopionionated in how you get there.

    Installation

    1. Add the dependency to your shard.yml:

      dependencies:
  tabular:
    github: leshaunj/tabular

    2. Run shards install

    Usage

    Ensure Tabular is imported into your project.

    require "tabular"

    Tokens

    A Tabular::Token represents any possible suggestion the end-user will receive, given their current argument. Tabular comes with three built-in Tabular::Token styles:

    require "tabular"

my_tokens = Tabular.tokens(
  Tabular.option("--opt", ["-o"], "A simple boolean flag"),

  Tabular.option("--arg-opt", ["-a"],
                 "A flag that expects any argument",
                 Tabular.argument(description: "Will suggest available files")),

  Tabular.option("--spec-arg", ["-s"],
                 description: "A flag that expects a choice of arguments",
                 Tabular.argument("choice1"),
                 Tabular.argument("choice2"),
                 Tabular.argument("choice3")),

  Tabular.option("--file-opt", ["-f"],
                 "A flag that expects a certain file type",
                 Tabular.argument("yaml", ["yml"], directives: :filter_ext)),

  Tabular.option("--dir-opt", ["-d"],
                 "A flag that expects any directory",
                 Tabular.argument(directives: :filter_dir)),

  Tabular.option("--dir-opt", ["-D"],
                 "A flag that expects any directory within a specific directory",
                 Tabular.argument("path/to/dir", directives: :filter_dir)),

  Tabular.command("action1", ["act1"], "A subcommand"),
  Tabular.command("action2", ["act2"], "Another subcommand"),
)

    [!tip] You can also create you own bespoke tokens with the base Tabular.Token class.

    Reply

    Once you have your Tabular::Tokens, invoke Tabular.reply when the __complete subcommand is called on your CLI. It will determine the suggestions and print them to STDOUT:

    require "tabular"

if ARGV[0]? == "__complete"
  ARGV.shift

  # Read arguments from `ARGV` or supply your own
  Tabular.reply my_tokens # , ["-a", "--bee", "cee", "dee"]
end

    If you provide a block to Tabular.reply, said block is called when a Tabular.command is matched, handing control back to your CLI:

    require "tabular"

if ARGV[0]? == "__complete"
  ARGV.shift

  Tabular.reply my_tokens do |current|
    case current.name
    when "subcmd1" then Subcommand1.new.complete ARGV
    when "subcmd2" then Subcommand2.new.complete ARGV
    end
  end
end

    Any unhandled Tabular.command tokens will fallback to default completion behaviour for the given shell.

    [!warning] When providing both a block and your own argument list, ensure your argument list is passed as a variable. Tabular.reply will modify this variable as needed so you can forward it to nested commands.

    Installer

    Call Tabular.install! when the completion (or what you choose) subcommand is called on your CLI:

    require "tabular"

if ARGV[0]? == "completion"
  ARGV.shift

  begin
    # Read arguments from `ARGV` or supply your own
    Tabular.install! # ["bash"]
  rescue e : Tabular::Error
    puts e
    exit 1
  end
end
